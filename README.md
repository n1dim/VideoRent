# VideoRent
Веб-приложение для управления видеопрокатом

---

# 1. Перечень и описание сценариев использования.
## 1.1 Роль: клиент
### Просмотр списка фильмов
- Пользователь открывает главную страницу.
- Система отображает список фильмов.
- Пользователь выбирает фильм.

### Просмотр карточки фильма
- Пользователь открывает страницу фильма.
- Система отображает:
   - название,
   - компанию,
   - режиссера,
   - год выхода,
   - описание
   - тип носителя,
   - стоимость,
   - количество доступных экземпляров.
  
### Регистрация
- Пользователь вводит ФИО, номер телефона, пароль и адрес.
- Система создает учетную запись.

### Авторизация
- Пользователь вводит номер телефона и пароль.
- Система выполняет вход.

### Работа с личным кабинетом
- Просмотр личных данных.
- Просмотр истории проката.
- Запрос продления аренды.
- Изменение личной информации.

## 1.2 Роль: Администратор
### Управление фильмами
- Добавление фильма
- Редактирование фильма
- Удаление фильма

### Выдача фильма
- Администратор выбирает фильм.
- Вводит номер телефона клиента.
- Указывает срок возврата.
- Система фиксирует выдачу.

### Возврат фильма
- Администратор выбирает фильм.
- Вводит номер телефона клиента.
- Фиксирует возврат.

### Управление пользователями
- Просмотр списка пользователей
- Блокировка / разблокировка
- Разрешение на продление срока аренды

# 4. Создание базы данных
```
CREATE TABLE Users (
    user_id SERIAL PRIMARY KEY,
    telephone_number VARCHAR(15) UNIQUE NOT NULL,
    full_name VARCHAR(100) NOT NULL,
    home_address VARCHAR(200) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(10) CHECK (role IN ('user', 'admin')) DEFAULT 'user',
    is_blocked BOOLEAN DEFAULT FALSE
);

CREATE TABLE movies (
    film_id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    film_director VARCHAR(100),
    description TEXT,
    company VARCHAR(100),
    release_year INT NOT NULL
);

CREATE TABLE media (
    media_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

CREATE TABLE copies (
    copy_id SERIAL PRIMARY KEY,
    film_id INT REFERENCES movies(film_id),
    media_id INT REFERENCES media(media_id),
    cost INT NOT NULL,
    count INT NOT NULL,
    status VARCHAR(20) CHECK (status IN ('available', 'unavailable'))
);

CREATE TABLE rentals (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    copy_id INT REFERENCES copies(copy_id),
    issue_date DATE NOT NULL,
    due_date DATE NOT NULL,
    return_date DATE,
    extension_requested BOOLEAN DEFAULT FALSE,
    requested_date DATE
);
```
